# Draft-website Skill

Use this skill to build a website draft for a potential customer. The goal is to create a dynamic, modern, and captivating experience with moving or flowing objects, while keeping the design grounded in strong business-specific best practices.

## What this skill should do

1. Ask the user what type of business the draft is for.
2. Ask if the business already has a website that can be referenced.
3. Ask whether they have a logo or social links to use as references.
4. Ask at least 5 styling and brand questions to avoid making websites look alike.
5. Search the web for best practices for website design in that business category, using FireCrawl or available web search tools if possible.
6. Produce a website draft that is suitable for GitHub hosting and easy to show to a potential customer.

## Questions to ask first

- What type of business is this for? (industry, service, location)
- Do they already have an existing website or web presence to reference?
- Do they have a logo, brand colors, or social media profiles I can review?
- What is the target audience for this website?
- What tone should the website convey? (e.g. professional, friendly, premium, fun)
- What key pages or sections should be included? (e.g. services, about, contact, testimonials)
- Are there any features or animations you want? (e.g. hero motion, floating elements, parallax, scroll transitions)
- What are the main goals of the website? (e.g. leads, bookings, sales, awareness)
- Are there any websites or designs you like that I can use as inspiration?

## How to use this skill

- Ask the user the questions above before generating the draft.
- If FireCrawl or a web search tool is available, search for “best website design practices for [business type]” and incorporate local business design patterns.
- Use the search results to suggest industry-specific section ideas and styling cues.
- Build a dynamic draft that includes moving or flowing objects, subtle animations, and a modern layout.
- Keep the result practical for a GitHub-hosted prototype: HTML/CSS/JS or a static site structure.

## Deliverables

- A short summary of the business and site goals.
- A page outline with sections and animation ideas.
- A style direction recommendation (colors, typography, motion).
- A draft website concept with sample content and dynamic behavior.
- Notes for deployment on GitHub Pages.

## FireCrawl guidance

If the FireCrawl CLI or web search capabilities are available, use them to research:
- Best homepage structure for this business type.
- Common service page patterns and local SEO elements.
- Design details for similar successful sites.

If FireCrawl is not available, use the best available web search tool.
